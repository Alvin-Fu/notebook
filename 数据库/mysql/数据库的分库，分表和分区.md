# 数据的拆分
数据拆分就是为了将单表的数据分散到多个数据库的多个表中

将数据尽可能的平均拆分，避免因为数据出现在单表中数据过大导致的**读写的性能问题**。
因为如果**不均匀**的拆分还是会产生数据访问++热点问题++， 热点数据增长过快会导致单表的性能问题
在使用中如何选择维度进行拆分：
	- 一般情况（大多的默认情况）：使用的是业务数据ID(一般是自增)进行拆分，这种情况下可以尽可能的平均
	- 更多的情况：是需要根据业务的情况进行拆分的
